Introduction

Validations allow us to set specific rules and limitations on what user are able to enter as data into specific areas. When something is entered that is not allowed, a message is shown to the user that provides information on what exactly they have entered incorrectly and how to fix it

Validations ensure that correct data is passed to the backend in the manner in which that data is required. It also makes sure that the forms are iodiot-proof for the user and as simple to use as possible

Required Validations

It is often neccassary to ensure that specific fields or sections of forms have been filled in before the form is submitted. For example, email, username or password in a login form

To make a field required, we use the "required" attribute on the form element

Best practice suggests marking each "required" form section with an asterix or other notable mark


Text Length Validations

Sometimes, the users will be restricted to a number of characters within the input of the form. This could be for a variety of reasons but one example would be Twitter/X with it's 140 character limit. Other uses would be to have a minimum or maximum character limit for passwords or usernames (longer usernames or password ensure greater security for example)

Minimum Length Validations

To provide a minimum length validation, we add the "minlength" attribute to the form control widget. Using an integer value will enforce a minimum character length

Maximum Length Validation

Similarly to requiring a minimum, length, we may need to restrict users to a mximum character length. In this circumstance, you would use the "maxlength" attribute with the form control

Combining Validations

You can apply as many validations as you like to a form control. With the tweet examples above, we can add a minimum and maximum length by combining the two attributes on the form control

Number Range Validations

Just the same as text-based length validation, sometimes we need to control the range of numbers that a user can enter into a number based form

We can use the "min" and "max" attributes again to set the lower and upper bonds of a form input

The "min" and "max" attributes only work with number based form controls such as number, dates and time inputs. The full list is available at this MDN page

Example uses for this type of validation would be a maximum or minimum order amount on a product order from or the number of passengers on a flight booking form

Min Validation

To add a minimum value validation, provide the form control a "min" attribute that uses an integer value which specifies the minimum number that form control will accept

Max Validation

To add a maximum validation, we provide the form control with an attribute of "max". This functions the same as the "min" value. An integer value is required to provide a maximum allowed amount. The "max" attribute is added to the form control to achieve this

Styling Validations

You can target for controls that have padded or failed validation using the :valid and :invalid pseudo-class
